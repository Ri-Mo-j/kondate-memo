# kondate-memo（献立メモ）

献立を整理するためのシンプルなメモアプリです。
登録した料理にタグを付けて管理できます。

このリポジトリは、アプリそのものに加えて、
**AIを活用した開発（AI駆動開発）の過程を記録すること**を目的にしています。

## デモ

ブラウザで動きます（スマートフォンでも動作します）。

https://ri-mo-j.github.io/kondate-memo/

## 機能

- 料理を名前とタグで登録
- タグは Enter で1つずつ確定、チップ表示・個別削除に対応
- 登録した料理を一覧表示
- データはブラウザの localStorage に保存（サーバー不要）

## なぜ作ったか

日々の献立を考えるのは、地味に手間がかかります。
「何を作ったか」「どんな料理があるか」を手元に整理しておきたい、
という素朴な動機から作り始めました。

同時にこのアプリは、**自分のAI駆動開発の練習台**でもあります。
小さく動くものを作り、少しずつ育てていく過程を、
そのまま記録として残しています。

## どう作っているか（AI駆動開発）

このアプリは、以下のサイクルで開発しています。

1. **人間が仕様を決める** — 何を作るか、どう振る舞うかを言葉にする
2. **AIが実装する** — 決めた仕様をもとにコードを生成
3. **人間が承認する** — 動作と内容を確認し、意図通りか・既存を壊していないかを判断

「コードを書く」より「何を作るかを決め、出てきたものを判断する」ことを、
人間側の主な仕事と位置づけています。
変更の理由は、できるだけコミットメッセージに残すようにしています。

## 技術メモ

- HTML / JavaScript の1ファイル構成
- 保存は localStorage のみ（バックエンド・ビルド不要）
- あえてシンプルな構成を保ち、段階的に拡張していく方針
